## 1. 내 컴퓨터들 간 사설 LAN (출구 없이)

**무엇을 얻는가** — 내 컴퓨터 몇 대를, 인터넷 어디에 있든, 하나의 암호화된
사설 LAN처럼 직접 연결.

**언제** — 가장 단순한 사용법. 인터넷 출구가 전혀 필요 없고, 그저 노트북·서버·
데스크탑끼리 SSH·파일 복사·내부 웹을 주고받고 싶을 때.

메쉬에 가입한 멤버는 자동으로 오버레이 IP `100.80.<메쉬id>.<멤버id>`를 받습니다.
예를 들어 메쉬 #1의 멤버 #2는 `100.80.1.2`. 평소 쓰던 도구를 그 IP로 그대로
쓰면 됩니다:

```sh
ssh user@100.80.1.2
scp file.tar.gz user@100.80.1.2:/tmp/
curl http://100.80.1.2:8080/
```

> 주의: 오버레이 MTU는 **1280**입니다. 출구를 따로 지정하지 않는 한 인터넷
> 트래픽은 평소대로 직접 나갑니다 — 이 레시피는 VPN이 아니라 사설 LAN입니다.

## 2. 풀터널 VPN / 지역 차단 우회

**무엇을 얻는가** — 내 모든 인터넷 트래픽을 다른 멤버를 통해 내보내서, 그 멤버의
공인 IP로 인터넷에 나가는 것처럼 보이게 함.

**언제** — 다른 나라에 있는 것처럼 보이고 싶을 때, 또는 지금 망에서 막힌 사이트에
접속하고 싶을 때.

출구가 될 멤버를 고른 뒤 풀터널을 켭니다:

```sh
lattice exit home alice     # alice(멤버)를 인터넷 출구로 지정
lattice vpn home            # 모든 트래픽을 그 출구로 라우팅
```

원래대로 직접 인터넷으로 되돌리려면:

```sh
lattice off
```

DNS와 라우팅은 자동으로 처리됩니다(`0.7.0`에서 수정). 터널 중 출구가 죽으면
**킬스위치**가 자동으로 직접 인터넷으로 되돌립니다.

> 주의: 풀터널인데 인터넷이 안 되면 구버전 빌드일 수 있습니다 — `meshd`를
> 재빌드하세요.

## 3. 한 컴퓨터에 여러 메쉬

**무엇을 얻는가** — 한 대의 컴퓨터를 동시에 여러 메쉬에 가입시켜, 서로 격리된
사설 네트워크를 함께 사용.

**언제** — 예를 들어 `work` 메쉬와 `home` 메쉬에 동시에 속하고 싶을 때. 각 메쉬는
자체 암호와 오버레이 대역 `100.80.<메쉬id>.x`를 가지며 서로 분리됩니다.

이 컴퓨터의 모든 메쉬를 한눈에:

```sh
lattice ls
```

각 메쉬에는 평소처럼 `lattice new` / `lattice join`으로 들어가면 됩니다.
풀터널(`lattice vpn`)은 한 번에 하나의 메쉬만 인터넷 출구로 쓸 수 있습니다.

> 주의: 멤버 id는 1바이트라서 **메쉬당 최대 254 멤버**입니다(`--max`로 더 낮게
> 제한 가능).

## 4. 휘발성 / 자폭 메쉬

**무엇을 얻는가** — 살아있는 멤버 수가 일정 시간 임계 아래로 떨어지면 메쉬 키가
스스로 폐기되는(복구 불가) 일회성 그룹.

**언제** — 잠깐만 쓰는 민감한 그룹. 모임이 흩어지면 흔적을 남기지 않게 하고 싶을 때.

생성 시 `--ephemeral`을 붙입니다:

```sh
lattice new secret --me alice --ephemeral
```

살아있는 멤버가 명단의 **60%**(`ceil(0.6*N)`) 아래로 grace(약 180초) 동안 머물면
메쉬 키가 폐기됩니다. 지금 메쉬가 영속인지 휘발성인지는:

```sh
lattice info secret      # charter 줄에 persistent / ephemeral 표시
```

> 주의: **기본은 꺼짐(persistent)** 입니다 — 노트북이 잠깐 잔다고 작은 메쉬가
> 터지는 사고를 막기 위해서입니다. 일부러 켤 때만 휘발성이 됩니다.

## 5. 공격 대응 (패닉 버튼)

**무엇을 얻는가** — 메쉬 전체에 즉시 경보를 울려, 막지 않으면 전원이 자폭하도록 함.

**언제** — 멤버나 기기가 탈취됐다고 판단될 때. 레시피 4(휘발성)와 무관하게 **항상**
사용할 수 있습니다.

아무 멤버나 경보를 울릴 수 있습니다:

```sh
lattice attack home
```

그러면 **생성자**가 짧은 grace(약 30초) 안에 all-clear를 보내지 않으면 전원 자폭합니다:

```sh
lattice allclear home    # 생성자만 — 자폭 취소
```

> 주의: one-veto · fail-deadly 설계입니다 — 한 명이라도 경보를 울리면, 생성자의
> 적극적인 취소가 없는 한 메쉬는 사라집니다. 신중히 쓰세요.

## 6. 키 교체 / 오프라인 멤버 퇴출 (re-cipher)

**무엇을 얻는가** — 메쉬 시크릿을 새것으로 갈아끼우고 epoch을 올려, 그 순간
오프라인이던 멤버를 사실상 퇴출.

**언제** — 떠난 멤버를 잘라내거나 주기적으로 키를 회전하고 싶을 때.

```sh
lattice recipher home
```

암호도 함께 바꾸려면:

```sh
lattice recipher home --cipher chachapoly-epoch
```

re-cipher 순간 오프라인이던 멤버는 새 트래픽을 열 수 없으며, 다시 들어오려면
재초대가 필요합니다.

> 주의: 실행하려면 명단의 **60% 이상이 온라인**이어야 합니다.

## 7. 암호(cipher) 선택

**무엇을 얻는가** — 메쉬의 데이터 플레인 암호를 선택.

**언제** — 메쉬를 새로 만들 때. 기본값으로 충분하지만, 연구용 암호를 시험해 보고
싶을 때 바꿉니다.

사용 가능한 암호 목록:

```sh
lattice ciphers
```

- `chachapoly-epoch` — 기본, 권장.
- `timewindow` — **실험적 연구용 암호**. 일반 사용에는 권장하지 않습니다.

생성 시 고정합니다:

```sh
lattice new home --me alice --cipher chachapoly-epoch
```

> 주의: 암호는 생성 시 고정됩니다. 이후 변경은 re-cipher(레시피 6)의
> `--cipher`로만 가능합니다.

## 8. 초대 비밀성 (알고리즘)

**무엇을 얻는가** — 초대장 위에 공유 비밀 계층을 한 겹 더 올려, 알고리즘을 따로
들은 사람만 초대장을 열 수 있게 함.

**언제** — 초대 코드가 새더라도, 별도로 약속한 알고리즘 이름을 모르면 가입하지
못하게 하고 싶을 때.

사용 가능한 invite-wrap 알고리즘 목록:

```sh
lattice algos
```

초대장을 알고리즘으로 감싸서 발급(호스트):

```sh
lattice invite home bob <id-code> --algo <알고리즘>
```

가입하는 쪽은 같은 알고리즘을 지정해야 열립니다:

```sh
lattice join <invite-code> --algo <알고리즘>
```

> 주의: 알고리즘 이름은 초대 코드와 **별도 경로(out-of-band)** 로 전달하세요.
> `--algo` 없이 쓰면 기본 동작 그대로입니다.

## 9. 재부팅·네트워크 변경에도 유지 (영속화)

**무엇을 얻는가** — 메쉬가 자동으로 디스크에 저장되고 데몬 시작 시 복원되어,
재부팅이나 Wi-Fi 변경에도 메쉬에서 빠지지 않음.

**언제** — 기본 동작입니다. 별도로 할 일은 없고, 동작을 조정하고 싶을 때만 환경변수를
씁니다.

데몬 실행 시 환경변수로 조정:

```sh
# 디스크 저장 끄기 (환경변수는 sudo 뒤에 — 앞에 두면 sudo가 떼어냄)
sudo DATA_PLANE=1 MESHD_NO_PERSIST=1 ./target/debug/meshd /tmp/lattice-meshd.sock

# 저장 위치 바꾸기
sudo DATA_PLANE=1 MESHD_STATE_DIR=/path/to/dir ./target/debug/meshd /tmp/lattice-meshd.sock
```

> 주의: 자폭(레시피 4·5)이나 `lattice rm` 시 디스크 사본도 함께 삭제됩니다.

## 10. 자동 피어 디스커버리 / NAT 통과

**무엇을 얻는가** — 피어 주소를 손으로 설정하지 않아도, 멤버들이 서로의 주소를
자동으로 학습.

**언제** — 거의 항상. 초대장, 약 20초 주기 가십, 공개 피어를 통한 반사 공인주소
학습, LAN 멀티캐스트 비콘으로 알아서 서로를 찾습니다.

대부분 추가 설정이 필요 없습니다. 다만 **공개 노드**(출구 서버 등)는 자신의 주소를
광고해 두면 남들이 찾기 쉽습니다 — 데몬 실행 시:

```sh
sudo DATA_PLANE=1 MESHD_BIND_PORT=41000 MESHD_ADVERTISE=<공인IP>:41000 ./target/release/meshd /tmp/meshd.sock
```

> 주의: 피어가 계속 `idle`이고 `live`가 안 되면 서로의 UDP 포트에 닿지 못하는
> 상태입니다 — 공개 출구는 `MESHD_ADVERTISE`를 설정하고 방화벽을 확인하세요.

## 11. 항상 켜져 있는 자동 보호

**무엇을 얻는가** — 따로 켤 필요 없이 기본으로 작동하는 보호.

별도 설정 없이 다음이 항상 동작합니다:

- 트래픽은 **헤더/본문 분리 암호**로 암호화됩니다.
- 와이어 프레임은 **핑거프린팅 저항**을 위해 스크램블됩니다.
- 풀터널에는 **킬스위치**가 있어, 출구가 죽으면 자동으로 직접 인터넷으로 복귀합니다.

> 주의: 전부 자동입니다 — 이 레시피에 해당하는 별도의 `lattice` 명령은 없습니다.

## 12. Lattice 업데이트 (메쉬 유지하면서)

**무엇을 얻는가** — 새 버전이 나오면 데스크탑 앱이 알려주고, 재설치 후에도 메쉬
가입 정보가 그대로 유지됨.

**언제** — 자동입니다. 업데이트가 있으면 앱 상단에 배너가 뜹니다.

- 앱 실행 시 GitHub Releases에서 새 버전을 확인합니다. 있으면 **Update** 배너가
  뜹니다(닫으려면 **Later**).
- **Update**를 누르면 먼저 모든 메쉬를 `<임시폴더>/lattice-mesh-backup.json`에
  백업한 뒤 다운로드 페이지를 엽니다. 새 버전을 설치하고 Lattice를 다시 열면, 새
  `meshd`가 그 백업을 다시 읽어 복원(이후 삭제)하므로, 설치 과정에서 로컬 상태가
  지워졌더라도 모든 메쉬에 그대로 남아 있습니다.
- CLI에서도 재설치 전에 같은 백업을 수동으로 만들 수 있습니다:

```sh
lattice backup                 # <임시폴더>/lattice-mesh-backup.json 으로 스냅샷
lattice backup /경로/x.json    # 또는 원하는 경로로
```

- **Windows 설치 프로그램**은 사용자별/시스템 전체 설치를 고를 수 있고, **MSI**는
  설치된 상태에서 다시 실행하면 변경 / 복구 / 제거 메뉴를 보여줍니다.

**주의:** 백업은 일회성 인계입니다 — `meshd`가 가져온 뒤 삭제합니다. 평소 영속화
(레시피 9)가 대부분의 업데이트를 이미 견디며, 이 백업은 상태 폴더를 지우는 깨끗한
재설치를 위한 안전망입니다.

## 곧 추가 예정 / 계획 (설계만 됨 — 아직 사용 불가)

아래 기능은 **설계 문서만 있고 CLI에는 아직 없습니다.** 지금은 사용할 수 없습니다.

- **메쉬 내부 DNS 이름** — 오버레이 IP 대신 이름으로 피어를 부르기.
  설계: [`../MESH_DNS.md`](../MESH_DNS.md). *CLI에 아직 없음.*
- **앱별 / 사이트별 스플릿 터널** — 특정 앱·사이트만 출구로 보내고 나머지는 직접
  내보내기. 설계: [`../AUTO_EXIT.md`](../AUTO_EXIT.md). *CLI에 아직 없음.*
